# Contributing to Human Exchange Protocol

Thanks for your interest in this project. Whether you want to fork it and build your own thing or work with me on the core protocol, you're welcome here.

## Fork it, build your own thing

The license allows it. No permission needed. If you have a vision for how proof-of-human verification or cooperative record-keeping should work, go build it. That's the point.

## Work with me on the core

If you want to contribute to this repo directly, here's how:

1. **Open an issue first.** Before writing any code, start a conversation. Describe what you want to change and why. This lets us align on whether it fits the project's direction before anyone invests time.

2. **Fork the repo and create a branch.** Work on your changes in your own fork. Name your branch something descriptive.

3. **Submit a pull request.** Reference the issue you opened. Explain what you did and how to test it. I'll review it and we'll discuss.

I'm the steward of this project, not the lead engineer. I care about the protocol's principles and direction. If you're a stronger coder than me, that's great. I want collaborators, not just contributors.

## Non-negotiables

These are the principles that define this protocol. Contributions that conflict with them won't be accepted.

- **Offline first.** The protocol must work without an internet connection on a basic smartphone. No exceptions.
- **No servers, no accounts.** Each participant maintains their own chain on their own device. Nothing phones home.
- **No commercialization.** This protocol is not a product. Contributions that introduce monetization, paywalls, premium tiers, or ad infrastructure will be rejected.
- **Proof of human, not proof of work.** The protocol verifies that two real people chose to record a cooperative act. It does not mine, stake, or speculate.
- **No third-party dependencies that phone home.** Every dependency must be auditable and must not transmit data to external services.

## No formal agreements required

For now, contributing is informal. By submitting a pull request, you agree that your contribution is made under the same AGPL v3 license as the rest of the project. No paperwork.

## Questions

Open an issue or email hello@humanexchangeprotocol.org. I'm happy to talk.
